#🤝 협업 규칙 (필독!)

프로젝트의 안정적인 관리와 충돌 방지를 위해 아래 규칙을 반드시 준수해 주세요.

1. 브랜치(Branch) 관리

main 브랜치 직접 작업 절대 금지: main은 언제나 실행 가능한 최종 버전만 유지합니다.

개인 브랜치에서 작업: 새로운 기능을 만들거나 에셋을 추가할 때는 반드시 본인 이름이 들어간 브랜치를 생성하세요.

규칙: develop-이름

    예시: develop-janghwan, develop-taehoon

2. 작업 완료 및 공유
자신의 브랜치에서 작업이 완료되면 **커밋(Commit) 후 푸시(Push)**를 진행합니다.

디스코드 공유: 작업물을 푸시한 후에는 디스코드 채널에 아래 내용을 공유해 주세요.

📸 스크린샷 또는 영상 (작업 내용 확인용)

📝 작업 내용 설명 (수정되거나 추가된 사항)

3. 코드 병합 (Merge)
공유 후에는 깃허브에서 main 방향으로 **Pull Request(PR)**를 생성합니다.

리드(또는 팀원)의 코드 리뷰와 승인이 완료된 후 main에 최종 병합됩니다.

📁 프로젝트 구조 안내

Assets/Textures: 원본 이미지 에셋 파일 관리 (임시)

Assets/Scripts: 모든 C# 스크립트 관리

Assets/Prefabs: 캐릭터, 함정, 아이템 등 모든 프리팹 관리

Assets/Sprites: 도트 에셋 및 이미지 리소스

Assets/Animations: 애니메이터 및 애니메이션 파일

⚠️ 주의사항

🔴 Scene 파일 관리
동일한 씬 파일을 동시에 수정하지 마세요. (깃 충돌 해결 불가)

개인 작업용 씬(Scene_이름)에서 먼저 기능을 구현하고 **프리팹(Prefab)**으로 만드세요.

완성된 프리팹을 메인 씬에 배치하는 방식으로 작업합니다.

🔴 Meta 파일 유실 방지
파일을 이동하거나 삭제할 때는 반드시 유니티 에디터 내부에서 진행하세요.

윈도우 탐색기에서 파일을 건드리면 .meta 파일이 꼬여 프로젝트가 깨질 수 있습니다.
